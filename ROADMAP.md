# Roadmap

This roadmap outlines possible directions for the project’s development.

The framework is already usable, but it is still evolving in terms of:
- methodology
- wording
- localisation
- examples
- interface quality
- community contribution

---

## Near-term priorities

### 1. Stabilise all current locales
- review all translations with native speakers
- ensure conceptual consistency across languages
- eliminate remaining localisation bugs
- unify terminology and tone

### 2. Improve documentation
- add extended methodology in English
- add extended methodology in Russian
- add FAQ
- document limitations of the framework
- explain how the 32 scenarios are derived

### 3. Improve interface quality
- refine browser locale detection
- improve multilingual rendering
- improve mobile layout
- reduce visual duplication in scenario cards
- improve accessibility and readability

### 4. Expand examples
- add more concrete task examples
- collect examples from design practice
- collect examples from teaching
- collect examples from media workflows
- collect “edge cases” where scenario selection is ambiguous

---

## Mid-term priorities

### 5. Build a case library
- user-submitted cases
- annotated use cases
- scenario comparison cases
- examples from education and workshops
- examples from real creative teams

### 6. Add methodological notes
- why these five dimensions were chosen
- what the framework can describe well
- what the framework does not capture
- how scenarios may shift during real workflows
- how the framework relates to authorship, control, and responsibility

### 7. Improve research usability
- add citation guidance
- add release versioning
- prepare archival releases
- connect the repository to Zenodo
- improve academic discoverability

---

## Longer-term directions

### 8. Comparative mode
Potential future features:
- compare two scenarios side by side
- compare neighbouring scenarios in the matrix
- explain why one scenario differs from another
- visualise transitions between scenarios

### 9. Printable and teaching versions
- printable matrix
- workshop handout
- teaching slides
- classroom discussion version
- facilitator guide

### 10. Community contributions
- create contribution templates
- open Discussions for methodology and examples
- invite native-speaker review of localisations
- invite case submissions from practitioners
- build a lightweight editorial workflow for accepted improvements

---

## Open questions

These areas remain open for discussion:
- whether five dimensions are sufficient
- whether some scenario titles should be made more neutral
- how strongly the framework should remain tied to creative industries
- whether to include sector-specific versions
- whether the framework should later support non-binary or weighted answers

---

## Contribution areas we especially welcome

We especially welcome help with:
- translation review
- wording refinement
- methodology critique
- educational use cases
- design and media case studies
- localisation QA
- accessibility improvements
